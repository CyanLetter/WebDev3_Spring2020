Grid

What is CSS Grid?

Making sure that anyone can use your site, under any condition.

Flexbox or Grid

Both, they work together with some overlap.

This has been a hot debate for years.

Use Grid For...

  • Building layout first
  • Larger page layouts
  • Controlling rows and columns
  • Overlapping elements

Grid properties are mostly applied to the parent.

Use Flexbox For...

  • Building content first
  • Small, one-dimensional components
  • Controlling rows or columns
  • Dynamic spacing between elements

No wrong answers

You will get more comfortable with practice. Use whichever will allow you to succeed quickly.

Let's code

Grid Template

Defining rows and columns


grid-template-columns: 100px auto 100px;
grid-template-rows: 80px auto 80px;

Fractional units

Automatically divide available space


grid-template-columns: 100px 2fr 1fr;

Grid gap

Shorthand for row-gap and column-gap


gap: 20px 10px;

Grid area

Naming rows and columns that containers might span


grid-template-columns: 100px auto 100px;
grid-template-rows: 80px auto 80px; 
grid-template-areas: 
  "header  header  header"
  "sidebar content related"
  "footer  footer  footer";

Child properties


grid-column-start: 1;
grid-column-end: span header;
// same as
grid-column: 1 / span header;

Same deal with grid-row

Remember, reordering your document's content through CSS can create accessibility problems.

Special functions

repeat()

Lets you build large consistent grids quickly.


grid-template-columns: 100px repeat(10, 1fr);

Makes named grid-areas tedious.

minmax()

Flexible grid sizes with constraints


grid-template-columns: minmax(80px 1fr) 3fr;

Putting it together

Dynamic responsive columns.


grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

That's it!

The basics at least.
